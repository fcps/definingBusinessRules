MECE Criterion

  • Mutually Exclusive
    • Everything in a computer is black and white (or 0/1), so rules must be as well.
    • For each rule there needs to be a consistent way to identify a single valid value.
    • It is not a rule if any records can fall into multiple categories simultaneously.
  • As an example, can a student have free and reduced price lunch status simultaneously?
  • If the answer is no, the condition is mutually exclusive.
  • If the answer is yes, you would need additional branches/nodes in the tree.
  • Completely Exhaustive
    • Every possible outcome must be accounted for.
    • This is the time where "What if" scenarios are extraordinarily valuable and helpful.
    • If there is a category/value that is not accounted for it is not a rule
  • There are three lunch statuses: Free, Reduced, and Full Price.
  • A student can only be in one of those three categories.
  • Because some DB developers want to take short cuts that degrade data quality, sometimes there are additional unmentioned rules.
  • For example, if there is no lunch status for a student it implies that they pay full price for their lunch.
  • The MECE criterion is the only standard.
  • Data managers do not define business rules.
  • Data managers can and will support you as Data Owners and Data Stewards to define effective business rules
  • Using this framework reduces confusion and improves data quality.
  • Emphasize role of data stewards and owners
  • It is difficult to be confused when there is a well defined bright line test for everything.
  • Data quality improves when it is clear what to enter and how to enter it at all times and under all conditions.

Pictures make rules easier to define and communicate

  • This shows the business rule for creating a new session ID for NWEA Map's Reporting Platform
  • The process starts when a user logs in to their system.
  • The login creates a new session ID to identify all the things that happen in that session.
  • Then we ask whether or not the user has logged out
  • If they have logged out that would indicate the end of the session and the last record with that session ID
  • If they haven't logged out, we ask if they've been active within the past 30 minutes.
  • If they have not been active in the last 30 minutes the NWEA platform logs the user out and ends the session
  • If they have been active in the last 30 minutes, we go back to checking if they've logged out.
  • No report views can be attributed to multiple session IDs for a single user and there are no cases that aren't covered. Either the user logs out, or the system logs them out after 30 minutes of no activity

How to define business rules

    • Student/Family answers home language survey
    • Student takes English language screener
    • If test score is below threshold student is identified as an English Learner
  • End to Beginning
    • Start with the known valid values
    • Students can be classified as not being identified for G/T programming, identified for G/T, or identified for Primary Talent Pool
    • If student is in grades K-3 they would be identified for primary talent pool
    • Did student have necessary evidence for formal identification
